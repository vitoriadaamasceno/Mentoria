# TDD - Test Driven Design

### Conceito

 È uma abordagem que tem como objetivo realizar um sistema 100% coberto por testes, sendo fundamental para evitar falhas futuras, porém nem sempre vai atingir 100%¹ mas boa parte dele.


### Objetivo

    A proposta é o colaborador vai escrever um teste automatizado antes de desenvolver o código.

### Passo a passo de como aplicar 

1. escreva um teste, mesmo que ainda não tenha em mãos o código real a ser testado, sem nada implementado;

2. realize o procedimento de teste e acompanhe o processo de falha;

3. produz-se o código real que deverá ser testado, de forma que ele passe no teste;

4. teste novamente, com o objetivo de passar no teste produzido;

5. revise o código, melhorando o que é preciso, e escreva-o por completo para testá-lo;

6. repita os passos de acordo com o que for necessário.
