### Tipos de Relacionamento


- Um para Muitos (One-to-Many): Normalmente, esse tipo de relacionamento resulta em uma lista de objetos. Por exemplo, um usuário pode ter vários endereços, então o relacionamento de um usuário para os seus endereços seria uma lista.

- Muitos para Um (Many-to-One): Esse relacionamento é geralmente representado por um único objeto (escalar). Por exemplo, muitos pedidos pertencem a um usuário específico.

- Muitos para Muitos (Many-to-Many): Esse tipo de relacionamento também é representado como uma lista de objetos. Por exemplo, um usuário pode participar de muitos grupos e cada grupo pode ter muitos usuários.
